# PRO-92: Public Profile with Badges

## Context

User profiles lack visual recognition of achievements. Badges gamify milestones (workouts completed, streaks, consistency) and drive engagement. This feature adds a badge system to the profile tab, gated behind a Firebase Remote Config feature flag.

## Phase 1: Domain Layer — Badge Entity + Rarity

### New file: `Domain/Entities/Badge.swift`

**BadgeRarity** enum (follows `LeagueTier` pattern):

- `common` (gray #6B7280), `rare` (blue #3B82F6), `epic` (purple #8B5CF6), `legendary` (gold #F59E0B)
- Properties: `displayName` (localized), `color` (hex), `sortOrder`

**BadgeType** enum (9 types for 3x3 grid):
| Type | Criteria | Rarity |
|------|----------|--------|
| `firstWorkout` | Complete 1 workout | common |
| `workouts50` | Complete 50 workouts | rare |
| `workouts100` | Complete 100 workouts | epic |
| `streak7` | 7-day streak | common |
| `streak30` | 30-day streak | rare |
| `streak100` | 100-day streak | legendary |
| `earlyBird` | 5 workouts before 7 AM | rare |
| `weekWarrior` | 7 workouts in a single week | epic |
| `monthlyConsistency` | Train every week for 4 consecutive weeks | rare |

Each has: `displayName`, `description` (criteria), `icon` (SF Symbol), `defaultRarity`

> Note: "1 ton lifted" badge dropped — `CompletedExercise` lacks weight/reps data. "Challenge winner" deferred — requires leaderboard coupling.

**Badge** struct:

```swift
struct Badge: Sendable, Identifiable, Codable, Equatable {
    let id: String  // BadgeType.rawValue
    let type: BadgeType
    let rarity: BadgeRarity
    let unlockedAt: Date?
    let isPublic: Bool
    var isUnlocked: Bool { unlockedAt != nil }
}
```

## Phase 4: UseCase — Badge Evaluation

### New file: `Domain/UseCases/BadgeEvaluationUseCase.swift`

Dependencies: `BadgeRepository`, `WorkoutHistoryRepository`, `FeatureFlagChecking`

**evaluate(userId:) -> [Badge]** (returns newly unlocked):

1. Check feature flag → return `[]` if disabled
2. Fetch existing badges from repository
3. Fetch workout history via `historyRepository.listEntries()`
4. Evaluate each `BadgeType` against history data
5. Save newly unlocked badges
6. Return new badges (for animation trigger)

**Evaluation data sources:**

- Workout count: `entries.count`
- Streak: computed from sorted dates (reuse logic from `ProfileProView.loadProfileData`)
- Early bird: `entries.filter { Calendar.current.component(.hour, from: $0.date) < 7 }.count >= 5`
- Week warrior: group by ISO week, check any week with 7+ entries
- Monthly consistency: 4 consecutive weeks each with >= 1 entry

### Test: `FitTodayTests/Domain/UseCases/BadgeEvaluationUseCaseTests.swift`

- Feature flag disabled → empty
- First workout unlocked when count >= 1
- Streak badges at correct thresholds
- Already unlocked → not duplicated
- Multiple badges unlocked in single evaluation
