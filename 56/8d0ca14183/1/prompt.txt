## Context

A UI do chat entre aluno e personal trainer ja esta pronta (`TrainerChatView`, `ChatBubble`, `TrainerDashboardView`), mas o backend e 100% mock — `TrainerChatViewModel` carrega mensagens hardcoded e o envio so adiciona ao array local. Esta issue cobre a integracao ponta a ponta com Firestore para mensagens em tempo real.

## Fase 1: Domain Layer — Protocolo e Entidades

### 1.1 Criar `TrainerChatMessage` (Domain Entity)

**Arquivo:** `Domain/Entities/TrainerChatMessage.swift`

```swift
struct TrainerChatMessage: Identifiable, Sendable, Hashable {
    let id: String
    let conversationId: String
    let senderId: String
    let content: String
    let type: MessageType
    let createdAt: Date
    let isRead: Bool

    enum MessageType: String, Sendable, Hashable {
        case text
    }
}
```

### 1.2 Criar `TrainerChatServiceProtocol`

**Arquivo:** `Domain/Protocols/TrainerChatServiceProtocol.swift`

```swift
protocol TrainerChatServiceProtocol: Sendable {
    func observeMessages(conversationId: String) -> AsyncStream<[TrainerChatMessage]>
    func sendMessage(conversationId: String, senderId: String, content: String) async throws
    func markMessagesAsRead(conversationId: String, byUserId: String) async throws
}
```

## Fase 2: Data Layer — Firebase Service + DTO + Mapper

### 2.1 Criar `FBConversation` e `FBChatMessage` (DTOs)

**Arquivo:** `Data/Models/FBTrainerChatModels.swift`

DTOs Firestore com `@DocumentID` e `@ServerTimestamp` seguindo o padrao existente em `FBPersonalTrainerModels.swift`.

Estrutura Firestore conforme issue:
```
conversations/{conversationId}
  ├── trainerId, studentId
  ├── lastMessage, lastMessageAt
  ├── unreadByTrainer, unreadByStudent
  └── messages/{messageId}
        ├── senderId, content, type, createdAt, isRead
```

`conversationId` deterministico: `"\(min(trainerId, studentId))_\(max(trainerId, studentId))"`

### 2.2 Criar `FirebaseTrainerChatService` (Actor)

**Arquivo:** `Data/Services/Firebase/FirebaseTrainerChatService.swift`

Seguindo o padrao de `FirebasePersonalTrainerService` e `FirebaseNotificationService`:

- `observeMessages(conversationId:)` — `AsyncStream` via `addSnapshotListener` na subcollection `messages/`
- `sendMessage(conversationId:, senderId:, content:)` — Escreve em `conversations/{id}/messages` + atualiza `lastMessage`, `lastMessageAt` e `unreadBy*` no doc pai. Cria `conversations/{id}` se nao existir (primeira msg)
- `markMessagesAsRead(conversationId:, byUserId:)` — Zera `unreadByStudent` ou `unreadByTrainer` conforme o userId

### 2.3 Criar `TrainerChatMessageMapper`

**Arquivo:** `Data/Mappers/TrainerChatMessageMapper.swift`

Mapeia `FBChatMessage` <-> `TrainerChatMessage` seguindo padrao de `PersonalTrainerMapper`.
