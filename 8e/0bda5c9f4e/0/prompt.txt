# PRO-101: Fix Permission Error on Personal Trainer Profile Access

## Context

Users receive a 403 error ("Você não tem permissão para acessar este recurso") when trying to view/search personal trainers. The error comes from the **CMS API** (REST), not do Firestore.

**Root cause**: O `CMSTrainerService.buildRequest()` envia SEMPRE um header `Authorization: Bearer {token}` em todas as requests — inclusive nos endpoints públicos como `GET /api/trainers`. Quando o Firebase ID token é inválido, expirado, ou não reconhecido pelo middleware de auth do CMS, o backend retorna HTTP 403. Porém, esses endpoints funcionam sem auth (verificado: `GET /api/trainers` retorna 200 sem header).

## Approach

Adicionar um parâmetro `requiresAuth` ao `buildRequest()` para que endpoints públicos (read-only) não enviem o header de autenticação. Endpoints que requerem autenticação (connect, reviews) continuam enviando.

## Changes

### 1. `CMSTrainerService.swift` (arquivo principal)

**Path**: `FitToday/FitToday/Data/Services/CMS/CMSTrainerService.swift`

Modificar `buildRequest()` (linha 165):

```swift
private func buildRequest(url: URL, method: String, requiresAuth: Bool = false) async throws -> URLRequest {
    var request = URLRequest(url: url)
    request.httpMethod = method
    request.setValue("application/json", forHTTPHeaderField: "Content-Type")
    request.setValue("application/json", forHTTPHeaderField: "Accept")

    if requiresAuth {
        if let tokenProvider {
            let token = try await tokenProvider()
            request.setValue("Bearer \(token)", forHTTPHeaderField: "Authorization")
        } else if let apiKey = configuration.apiKey {
            request.setValue("Bearer \(apiKey)", forHTTPHeaderField: "Authorization")
        }
    }

    return request
}
```

Marcar endpoints que necessitam auth com `requiresAuth: true`:

- `requestConnection()` (linha 101) — POST, precisa de auth
- `checkConnectionStatus()` (linha 116) — GET, precisa de auth (dados do aluno)
- `submitReview()` (linha 129) — POST, precisa de auth

Endpoints públicos ficam com default `false` (sem mudança nas chamadas):

- `fetchTrainerReviews()` — reviews públicos
- `fetchPrograms()` / `fetchProgram(id:)` — programas públicos
