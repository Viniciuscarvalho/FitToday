4. Errors and fixes:

   - **Compilation error in FirebaseFeedService.swift line 83**: `optional type 'Any?' cannot be used as a boolean; test for '!= nil' instead` and `invalid conversion from throwing function type` in the `toggleLike` method's `db.runTransaction` closure.
     - **Root cause**: Firestore's `runTransaction` expects a non-throwing closure returning `Any?`, but I had a throwing closure returning `Bool`
     - **Fix**: Changed to use `errorPointer` pattern for error handling (non-throwing closure), box `Bool` as `NSNumber`, and unbox the result after the transaction:
     ```swift
     let result: Any? = try await db.runTransaction({ transaction, errorPointer in
         let snapshot: DocumentSnapshot
         do {
             snapshot = try transaction.getDocument(docRef)
         } catch let error as NSError {
             errorPointer?.pointee = error
             return nil
         }
         // ... logic ...
         return NSNumber(value: isNowLiked)
     })
     return (result as? NSNumber)?.boolValue ?? false
     ```
   - **Multiple Swift 6 concurrency warnings** in FirebaseFeedService.swift about main actor-isolated conformances of FBFeedPost/FBFeedComment to Codable being used in actor-isolated context - these are warnings (not errors) that will become errors in Swift 6 mode. Not yet fixed.
