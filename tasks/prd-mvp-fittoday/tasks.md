# Resumo de Tarefas de Implementação de MVP FitToday

## Tarefas

- [ ] 1.0 Fundação do app: camadas (sem módulos), Swinject DI, Router/DeepLinks e TabBar com stacks independentes (L)
- [ ] 2.0 Design System SwiftUI (inspirado no kit) + componentes base reutilizáveis (M)
- [ ] 3.0 Domain: modelos, protocolos de repositório e UseCases (M)
- [ ] 4.0 Data: SwiftData (perfil/histórico/pro) + loader JSON de blocos + mappers + repos concretos (L)
- [ ] 5.0 Onboarding (3 telas) + Setup (questionário inicial com stepper) (L)
- [ ] 6.0 Home (“Treino de Hoje”) + estados de jornada (sem perfil / respondeu hoje / free/pro) (M)
- [ ] 7.0 Questionário diário (2 passos) + validações + roteamento para paywall/geração (M)
- [ ] 8.0 Motor de treino: combinação de blocos curados + ajuste por dor + geração local (L)
- [ ] 9.0 Treino gerado: lista + detalhe do exercício + fluxo de execução + conclusão (L)
- [ ] 10.0 Histórico: lista por dia + status concluído/pulado (M)
- [ ] 11.0 Biblioteca Free: treinos fixos por objetivo/estrutura + iniciar treino (M)
- [ ] 12.0 Pro: Paywall + StoreKit 2 (assinatura) + restore + entitlements + gating (L)
- [ ] 13.0 Testes: unitários (Domain/Data) + UI tests (fluxos críticos) + StoreKit Testing (M)
- [ ] 14.0 Performance e acessibilidade: auditoria SwiftUI + otimizações + polish (M)

## Notas sobre tamanho
- S - Small
- M - Medium
- L - Large
